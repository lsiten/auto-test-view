# TASK-007-02: PageIndex Service 懒加载 + 去除 litellm

## 父任务
TASK-007

## 涉及文件
- `electron/recorder/semantic-index.ts`（改造）
- `lib/pageindex/pageindex/utils.py`（改造）
- `lib/pageindex/requirements.txt`（改造）
- `electron/main.ts`（改造 PageIndex 相关部分）

## 方案要点
1. `semantic-index.ts` 改造：
   - `startPageIndexService()` 改为 `setPageIndexConfig(config)` 仅存储配置
   - 新增 `ensurePageIndexService()` — 首次调用时 spawn Python 进程
   - 所有公开 API（indexRecording, getDocumentList 等）内部调用 `ensurePageIndexService()`
   - `callLlm()` 改为直接 HTTP 调用 LLM 服务（通过 `ensureLlmService()` 获取 URL）
   - `isPageIndexAvailable()` 改为检查 config 是否已设置 + 脚本是否存在
2. `utils.py` 改造：
   - 移除 `import litellm`、`import pymupdf`、`import PyPDF2`
   - `llm_completion()` / `llm_acompletion()` 改为通过 HTTP 调用 `OPENAI_API_BASE`
   - `count_tokens()` 改为简单估算（`len(text) // 3` 中文，`len(text) // 4` 英文）
   - 移除所有 PDF 相关函数（本项目仅用 MD 索引）
3. `requirements.txt` 精简为 `pyyaml` + `python-dotenv`
4. `main.ts` 移除 `startPageIndexService(llmConfig)`，改为 `setPageIndexConfig(llmConfig)`

## 验收标准
1. 启动时不 spawn PageIndex Python 进程
2. 首次索引/查询时自动启动 PageIndex 服务
3. Python 进程不再加载 litellm/pymupdf/PyPDF2
4. 录制索引和匹配功能正常
5. TypeScript 编译通过
