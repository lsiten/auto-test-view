# TASK-005-01: Store 双路径改造

## 父任务
TASK-005

## 涉及文件
- `electron/recorder/store.ts`

## 方案

### 新增类型
```typescript
type RecordingScope = "project" | "global";
```

`RecordingIndexEntry` 增加 `scope: RecordingScope` 字段，标识录制来源。

### 路径管理
新增模块级变量：
```typescript
let projectRecordingsDir: string | null = null;  // <cwd>/.auto-test-view/recordings
let globalRecordingsDir: string;                  // ~/.auto-test-view/recordings
```

- `initRecorderStore(projectDir?: string)` — 设置两个路径，创建目录
- 内部 helper `getRecordingsDir(scope)` 按 scope 返回对应目录
- 内部 helper `getIndexPath(scope)` / `getRecordingPath(id, scope)` 同理

### CRUD 改造
- `saveRecording(recording, scope)` — scope 决定保存到哪个目录
- `listRecordings(group?, scope?)` — 无 scope 时合并两个目录的结果（project 在前）
- `getRecording(id, scope?)` — 无 scope 时先查 project，再查 global
- `deleteRecording(id, scope?)` — 同上查找逻辑
- `updateRecording(id, data, scope?)` — 同上
- `searchRecordings(query, scope?)` — 无 scope 时合并搜索
- `generateRecordingId(scope)` — 按 scope 对应 index 生成
- `exportRecording`、`deleteStep`、`updateStep`、`batchXxx` — 加 scope 参数

### 向后兼容
- scope 参数可选，默认行为 = 先 project 后 global
- 无 projectDir 时退化为仅 global

## 验收标准
1. `initRecorderStore()` 支持传入 projectDir，创建 global + project 两个目录
2. 不传 projectDir 时退化为仅 global 模式
3. `listRecordings()` 合并两个 scope 的录制，每条带 `scope` 字段
4. `getRecording(id)` 先查 project 再查 global
5. `saveRecording(rec, "project")` 保存到项目目录
6. `saveRecording(rec, "global")` 保存到全局目录
7. TypeScript 编译通过
