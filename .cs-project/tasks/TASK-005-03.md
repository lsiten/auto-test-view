# TASK-005-03: IPC/Preload/Main/MCP 接入 scope

## 父任务
TASK-005

## 依赖
TASK-005-01 (Store 改造完成)

## 涉及文件
- `electron/core/ipc-handlers.ts`
- `electron/preload.ts`
- `electron/main.ts`
- `electron/mcp/server.ts`

## 方案

### main.ts
- 启动时确定 project 目录：`process.env.AUTO_TEST_PROJECT_DIR || process.cwd()`
- `initRecorderStore(projectDir)` 传入项目目录
- `startPageIndexService(llmConfig, projectDir)` 传入项目目录

### ipc-handlers.ts
- 所有 `recorder-*` IPC handler 接收可选 scope 参数并透传给 store
- `recorder-list` → `listRecordings(group, scope)`
- `recorder-get` → `getRecording(id, scope)`
- `recorder-delete` → `deleteRecording(id, scope)` + `removeProfile(id, scope)`
- 新增 `recorder-save` 接收 scope
- `record-control` stop 时新增 scope 参数控制保存位置
- `semantic-index-rebuild` 可指定 scope

### preload.ts
- `recorderStore.list(group?, scope?)` — IPC 传 scope
- `recorderStore.save(rec, scope)` — 新增
- 其他操作加可选 scope 参数
- 新增 `getProjectDir(): Promise<string | null>` — 查询当前项目目录

### mcp/server.ts
- recording 相关 MCP 工具加可选 `scope` 参数（z.enum(["project", "global"]).optional()）
- `start_recording` 加 scope 参数
- `list_recordings` 加 scope 过滤
- `execute_task` 中 matchRecording 逻辑不变（matcher 内部已处理优先级）

## 验收标准
1. main.ts 启动时正确确定项目目录并传给 store
2. 所有 IPC handler 支持 scope 参数透传
3. preload 暴露 scope 参数
4. MCP 工具支持 scope 参数
5. TypeScript 编译通过
