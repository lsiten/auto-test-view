## 方案要点

### network-interceptor.ts（规则引擎）
- 内存维护活跃拦截规则列表 InterceptRule[]
- 每条规则：id, urlPattern(glob), resourceType?, method?, action(mock|block|modify|delay|fail), 对应参数
- 规则变更时自动调用 CDP Fetch.enable 更新 requestPatterns
- 监听 CDP Fetch.requestPaused 事件，按规则匹配执行：
  - mock → Fetch.fulfillRequest（自定义 responseCode + headers + body）
  - block → Fetch.failRequest(reason: "BlockedByClient")
  - modify → Fetch.continueRequest（修改请求头后放行）
  - delay → setTimeout 后 Fetch.continueRequest
  - fail → Fetch.failRequest（指定 errorReason）
- 无规则匹配时 Fetch.continueRequest 放行
- 页面导航后自动重新注册（监听 CDP Client 的 re-attach 事件）

### network_log（流量捕获）
- CDP Network.enable 开启网络追踪
- 监听 Network.requestWillBeSent + Network.responseReceived + Network.loadingFinished
- 内存维护请求日志（限最近 500 条）：url, method, status, headers, timing
- 支持 start/stop/get/clear 操作
- get 支持 filter（urlPattern, method, statusCode）

### MCP 工具
- network_intercept: action(add|remove|list|clear) + rule 参数
- network_log: action(start|stop|get|clear) + filter 参数

## 验收标准
1. network_intercept 支持 add/remove/list/clear 四种操作
2. mock 动作能返回自定义响应体和状态码
3. block 动作能阻止请求
4. modify 动作能修改请求头后放行
5. delay 动作能延迟指定时间后放行
6. fail 动作能模拟网络错误
7. 页面导航后拦截规则自动重新注册
8. network_log 能捕获、过滤、清除网络流量
9. 两个工具产生录制事件
