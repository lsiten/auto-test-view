## 方案要点

### upload_file（文件选择自动化）
- 参数：filePaths: string[], selector?: string
- CDP DOM.getDocument → DOM.querySelector 定位 <input type="file">
- 无 selector 时自动查找页面中第一个 file input
- CDP DOM.setFileInputFiles 注入文件路径
- 通过 Runtime.evaluate 触发 change 事件

### drag_file（拖拽上传自动化）
- 参数：filePaths: string[], selector: string
- 主进程读取文件内容 → base64 编码
- executeJavaScript 注入页面：base64 → Blob → File 对象 → DataTransfer
- 合成 dragenter → dragover → drop 事件，派发到目标元素

### showOpenDialog 待注入文件队列
- main.ts 中维护 pendingUploadFiles: string[] 队列
- upload_file 调用时，先将文件路径入队
- showOpenDialog 拦截逻辑：有队列文件时返回队列文件，否则静默取消
- 暴露 setPendingUploadFiles / clearPendingUploadFiles 接口

## 验收标准
1. upload_file 通过 CDP 自动注入文件到 <input type="file">，触发 change 事件
2. drag_file 通过合成 DragEvent 将文件投递到 Drop zone，触发 drop 事件
3. showOpenDialog 有待注入文件时自动返回，否则静默取消
4. 两个工具执行过程中零人工交互
5. 两个工具产生录制事件
