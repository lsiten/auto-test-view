## 验收标准
1. InstanceManager 类导出，支持以下操作：
   - `start()` — 启动管理器，创建最少数量的实例
   - `acquire()` — 获取空闲实例（无空闲时扩容或等待）
   - `release(instanceId)` — 释放实例回池
   - `shutdown()` — 关闭所有实例
2. 实例通过 `child_process.spawn` 启动 Electron，传入 `MCP_PORT` 环境变量
3. 端口分配：从 3401 开始递增，每个实例不同端口
4. 最大实例数 = `os.cpus().length`，最小实例数 = 1
5. 空闲检测：实例 release 后开始计时，超过 IDLE_TIMEOUT_MS（默认 5 分钟）自动关闭
6. 实例崩溃（进程退出）自动重启（如果当前数量 < 最小数）
7. 健康检查：通过 HTTP GET 确认实例 MCP 端口可达
8. esbuild 构建通过

## 方案要点

### types.ts
```typescript
interface PoolConfig {
  readonly minInstances: number;    // default: 1
  readonly maxInstances: number;    // default: os.cpus().length
  readonly idleTimeoutMs: number;   // default: 5 * 60 * 1000
  readonly basePort: number;        // default: 3401
  readonly healthCheckIntervalMs: number; // default: 30000
}

type InstanceStatus = "starting" | "ready" | "busy" | "stopping";

interface InstanceInfo {
  readonly id: string;
  readonly port: number;
  readonly pid: number;
  status: InstanceStatus;
  lastActiveAt: number;
}
```

### instance-manager.ts
- 维护 `Map<string, InstanceInfo>` + 对应的 `ChildProcess` 引用
- `spawn('electron', ['dist/electron/main.js'], { env: { ...process.env, MCP_PORT: port } })`
- 启动后轮询 health check 直到 ready
- acquire 时优先分配最近活跃的 ready 实例（热缓存）
- 空闲定时器：每个实例 release 后 setTimeout，acquire 时清除
- 进程退出监听：`child.on('exit')` 触发重启逻辑
