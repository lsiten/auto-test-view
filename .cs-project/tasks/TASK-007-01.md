# TASK-007-01: LLM Service 懒加载改造

## 父任务
TASK-007

## 涉及文件
- `electron/core/llm-service.ts`（新建）
- `electron/core/llm-proxy.ts`（改造）
- `electron/main.ts`（改造 LLM 相关部分）

## 方案要点
1. 新建 `llm-service.ts` 统一管理 LLM 代理生命周期：
   - `setLlmServiceConfig(config)` — 仅存储配置
   - `ensureLlmService()` — 首次调用时启动 llm-proxy，返回 proxyBaseUrl
   - `stopLlmService()` — 关闭代理
   - 内部维护 singleton 状态
2. `llm-proxy.ts` 保持现有逻辑不变，仅被 llm-service.ts 调用
3. `main.ts` 中 `resolveLlmConfig()` 不再启动 proxy：
   - 移除 `await startLlmProxy(apiKey, rawBaseUrl)`
   - 改为 `setLlmServiceConfig({ apiKey, rawBaseUrl, model })`
   - 将原始配置（非 proxy URL）传递给各模块
   - page-agent 注入时通过 `ensureLlmService()` 获取实际 URL

## 验收标准
1. 启动时不启动 llm-proxy HTTP server
2. page-agent 首次 LLM 调用时自动启动 proxy
3. 多次调用 `ensureLlmService()` 复用同一实例
4. `stopLlmService()` 正确关闭
5. TypeScript 编译通过
