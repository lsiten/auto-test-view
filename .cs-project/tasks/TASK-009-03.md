## 验收标准
1. 代理服务器监听端口 3399（或 `POOL_PORT` 环境变量）
2. MCP initialize 请求：acquire 实例 → 代理到实例端口 → 返回响应，同时记录 session→instance 映射
3. 后续请求（带 mcp-session-id）：根据映射路由到对应实例
4. session 关闭（DELETE /mcp）：释放实例回池
5. 请求代理透传所有 headers（包括 mcp-session-id），SSE 响应流式转发
6. `npm run start:pool` 启动代理服务器（非 Electron，普通 Node 进程）
7. 启动时自动创建最少数量的 Electron 实例
8. esbuild 构建通过
9. 手动测试：`curl` 发 initialize 请求到 3399 能正常返回

## 方案要点

### proxy-server.ts
- 普通 Node HTTP server（不是 Electron 进程）
- 维护 `Map<sessionId, instanceId>` 映射
- initialize 请求流程：
  1. `instanceManager.acquire()` 获取实例
  2. 代理请求到 `127.0.0.1:<instance.port>/mcp`
  3. 从响应中提取 mcp-session-id
  4. 建立 sessionId → instanceId 映射
  5. 返回响应给调用者
- 非 initialize 请求：从 header 取 sessionId → 查映射 → 代理到对应实例
- SSE 响应：pipe 转发，不做缓冲
- session 超时 / DELETE：释放实例

### package.json
```json
"start:pool": "node dist/electron/pool/proxy-server.js"
```

### esbuild 适配
- proxy-server.ts 需要作为独立入口打包（非 Electron 进程）
